<p>To follow these standards, the bean needs one or more <i>properties</i> abstractions which represent different state values of an object. A property has a name (a valid Java identifier) and a type (either a <a href="/wiki/Java_Programming/Classes,_Objects_and_Types" title="Java Programming/Classes, Objects and Types">reference type</a>, a <a href="/wiki/Java_Programming/Primitive_Types" title="Java Programming/Primitive Types">primitive type</a> or an <a href="/wiki/Java_Programming/Arrays" title="Java Programming/Arrays">array type</a>). By default, the properties of a JavaBean class are inferred by the presence of either a getter method, a setter method, or both:</p> 
<ul> 
 <li>A getter method which is used to obtain the value of the property from a bean. The name is usually of the form <code>get<i>PropertyName</i></code>. For example, the getter method for the String property <code>whiskey</code> is <code>public String getWhiskey()</code>. For boolean properties (those whose type is <a href="/wiki/Java_Programming/Keywords/boolean" title="Java Programming/Keywords/boolean">boolean</a>, the convention is to use the naming pattern <code>is<i>PropertyName</i></code>.</li> 
</ul> 
<p><code>isDiscounted()</code> would be the getter method for a boolean property named <code>discounted</code>. Thus, the method signature of most getters is <code>public <i>PropertyType</i> get<i>PropertyName</i>()</code> or <code>public boolean is<i>PropertyName</i>()</code>.</p> 
<ul> 
 <li>A setter method which is used to assign a value to a bean's property. A setter method is a method of the form <code>public void set<i>PropertyName</i>(<i>PropertyType</i> value)</code>. For the previous example, the setter could be invoked as <code>setWhiskey("bourbon");</code>.</li> 
</ul> 
<p>As per the JavaBeans standard, getters and setters defined as described above automatically determine the properties of the class. However, by creating <code>java.beans.PropertyDescriptor</code> classes, you can specify alternate implementations by explicitly declaring the property names and the getter and/or setter methods for each property.</p> 
<p>Properties are often implemented with <a href="/wiki/Java_Programming/Scope" title="Java Programming/Scope">private</a> <a href="/wiki/Java_Programming/Variables" title="Java Programming/Variables">instance variables</a>, but this is not required.</p>
